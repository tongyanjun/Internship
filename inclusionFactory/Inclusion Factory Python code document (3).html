Inclusion Factory Problem Description and Python Code¶

这个文档作为Inclusion Factory项目的内部文档，是用来说明排班系统的问题描述，解决算法，以及如何在Python中实现的，将分为四个部分：

我们将在这个章节介绍Inclusion Factory的背景，以及需要考量的点。

背景介绍与初步思路¶

首先我们了解到，IF目前雇员为22名，均为有不同程度的智力与肢体障碍的人士，大部分不识字。工人在工厂中从事主要由德资车企提供的简单地拼装工作，也就是又客户向工厂提供元件，由工厂进行拼装生产后发货给客户，采取订单制管理模式，由job coach或manager进行每天的排班。主要生产的产品有十余种，每种产品可能包含多个工序，存在较为简单地生产逻辑，也就是简单地流水线式作业。例如：

  • 行程开关
    • 摆螺丝
    • 组装开关
  • 安全阀
    • 预放组件
    • 推安全阀
    • 检查

每一个工人每天进行四个shift的生产工作，每一个shift可以进行不同工序的生产，也可以一天内四个shift都为同一种工序生产。所以该问题可以抽象为变量为0/1的线性规划排班问题，我们将在下一个章节中给出数学表达。

Inclusion Factory的问题描述与数学表达¶

我们提到，IF的诉求为在每个工人每天至多四个班次的情况下，进行科学的排班，以完成接受的订单。所以，我们可以抽象为线性规划排班问题，也就是给定一个目标函数与诸多限制条件，寻找最优的可行解。目标函数可以根据我们的需求给出，例如最小化总工时等于按照各个订单的deadline组织生产，最大化预计产出等于全产能生产等。限制条件为客观条件，例如每人每天总班次不得超过4，生产的产品数量必须大于最低要求，同一个产品的不同生产工序要保证上游产能略大于下游产能等。

所以我们将在后面逐步讨论：

  • 数据与数据预处理
  • 问题的数学表达
  • 将来拓展

数据与数据预处理¶

生产力数据(Worker efficiency)

由于每个工人接受的训练不同，身体情况与精神状况的不同，导致各个工人所能生产的产品与生产效率不尽相同，下表为各个工人对于目前工厂生产的产品的生产效率，即每小时所生产的产品数量。

因为上表中采取的是基于历史数据的平均值，在实际使用中会出现较大的偏差与风险。据工厂方介绍，每个工人的生产效率可能会因为情绪影响，偏差达到一倍。故我们在实际使用中采取历史数据的百分之四十分位数，保证了排班算法的稳健性与保守性。

问题的数学表达¶

首先我们先定义数据中的数学符号(Notation):

  • $W_i$ ：工人i的ID，i的取值范围为 $0-21$，共计22名工人，最大值记为$W_{max}$
  • $P_i$ ：产品工序i的ID，i的取值范围为 $0-12$，共计13种工序，最大值记为$P_{max}$
  • $E_{ij}$：工人i对产品工序j的生产效率，即每小时所能生产的数量
  • $D_i$ ：产品工序i的剩余工期
  • $O_i$ ：产品工序i的待生产数量

以及我们输出的结果变量为：

  • $V(w, s, p)$ ：类型为Boolean, 值为0/1，表示当天在岗工人$w$是否在$s$班次生产产品$p$
  • $R_{low}(p)$ ：类型为Integer，表示当天产品工序$p$的预计最少数量
  • $R_{up}(p)$ ：类型为Integer，表示当天产品工序$p$的生产上限数
  • $EstOut(p)$ ：类型为Integer，表示当天产品工序$p$的预计生产数量

其中，设置$R_{low}(p)$与$R_{up}(p)$的原因是保证工厂生产在合理的区间内运行，也就是通过设置线性规划中的限制条件，来保证我们的预估产量在一个合理的区间之内。设置最低的生产数量是为了可以在订单截止之前完成生产，而设置最大生产数量，是为了让各个产品均衡生产，防止出现个别产品被过度生产，而其余的产品仅仅只满足最低要求。所以，$R_{low}(p)$与$R_{up}(p)$的定义为：

$$ R_{low}(p) = \frac{O_p}{D_p} $$$$ R_{up}(p) = 5\times\frac{O_p}{D_p} $$

这里设置上限为5倍的最低产量的原因是通常考虑到订单周期通常大于5，一方面保证了可行域范围存在，另一方面保证了不会出现超出订单要求的产出。

优化目标函数

我们还是以注重员工工作多样性的最大生产排产为例。这里是我们首先要考量的目标：

  • 希望排产的产出为最大，在满足各个限制条件的情况下
  • 每个工人每个班次只能生产一种产品
  • 对于任意生产工序$p$，每日的预计产量不得少于$R_{low}(p)$
  • 对于任意生产工序$p$，每日的预计产量不得大于$R_{up}(p)$
  • 每个工人每天的劳动班次总数不得超过4
  • 对于任意两个工人，当日的劳动班次总数相差不得超过1
  • 上游生产工序的产能略大于下游生产工序的产能
  • 每个员工当日至少从事两种以上的任务

所以，我们要最大化的目标函数为：

$$ \max{Z} = \sum_{p=0}^{P_{max}}\frac{N(p) - R_{low}(p)}{R_{low}(p)} $$

其中$N(p)$是当前排班下的产品工序$p$的预计总产出，目标函数代表了每个产品工序实际排产的产量超出最低生产标准的比率，通过最大化$Z$来确保我们的排产是最大化产出的排产。

限制条件

我们需要一些客观现实的限制条件来给出问题的可行域，也就是说我们的最优化结果是在下面的限制条件都满足的情况下的最优结果。

  • 每个工人每个班次只能生产一种产品

$\quad \ \;$因为我们的变量为$V(w, s, p)$，代表工人$w$在班次$s$是否从事生产工序$p$的生产工作，为0/1变量，所以对于任意工人$w$和班次$s$，我们有：

$$ \sum_{p = 0}^{P_{max}} V_{w,s,p} \le 1 $$
  • 对于任意生产工序$p$，每日的预计产量不得少于$R_{low}(p)$
$$ \sum_{w = 0}^{W_{max}} \sum_{s = 0}^{3} \sum_{p = 0}^{P_{max}} V_{w,s,p} \times E(w,p) \ge R_{low}(p) $$
  • 对于任意生产工序$p$，每日的预计产量不得多于$R_{low}(p)$
$$ \sum_{w = 0}^{W_{max}} \sum_{s = 0}^{3} \sum_{p = 0}^{P_{max}} V_{w,s,p} \times E(w,p) \le R_{up}(p) $$
  • 每个工人每天的劳动班次总数不得超过4

$\quad \ \;$对于任意一个工人$w$，我们有：

$$ \sum_{s = 0}^{3} \sum_{p=0}^{P_{max}} V(w,s,p) \le 4 $$
  • 对于任意两个工人，当日的劳动班次总数相差不得超过1

$\quad \ \;$这里是为了保证不会出现能力较强的工人班次排满，而能力较弱的工人出现过多地"No Task"情况，对于任意两个工人$w_i$和$w_j$，我们有：

$$ | \sum_{s = 0}^{3} \sum_{p=0}^{P_{max}} V(w_i,s,p) - \sum_{s = 0}^{3} \sum_{p=0}^{P_{max}} V(w_j,s,p) | \le 1 $$
  • 上游生产工序的产能略大于下游生产工序的产能

$\quad \ \;$这里是为了保证下游的生产线不会因为上游半成品供应不足而导致停产，如果考虑半成品的历史存货，则为：对于任意一组上游产品$p_i$与下游产品$p_j$，和任意班次$s$：

$$ \frac{(O_i - O_j)}{4} + \sum_{w = 0}^{W_{max}} V_{w,s,p_i} \times E(w,p_i) \ge \sum_{w = 0}^{W_{max}} V_{w,s,p_j} \times E(w,p_j) $$

$\quad \ \;$上式的含义是指上游产品对比下游产品的绝对盈余除以一天的班次总数加上本班次生产的上游产品工序，要大于下游产品工序的产量。

  • 每个员工当日至少从事两种以上的任务

$\quad \ \;$这里是出于对员工发展的考量，希望强制每个员工从事多种不同的工作来熟悉各个类型的技能，所以对于任意工人$w$和任意产品工序$p$，我们有：

$$ \sum_{s = 0}^{3} V(w,s,p) \le 3 $$

$\quad \ \;$上式通过限制每个工人分配到同种产品工序的班次总数上限来确保每个工人至少从事两种以上的工作。

解决算法与package¶

为了解决上面的线性规划问题，我们需要使用单纯型法，也就是著名的大M算法。而通过调用python的pulp包裹，我们可以定义自己的线性规划问题，并进行快速求解。对于pulp包裹的详细介绍见：https://pythonhosted.org/PuLP/

简言之，pulp的使用步骤包括：

  • 相关包裹载入
  • 数据的读取（HANA数据库或包含json文件的service）
  • 数据预处理
  • 定义线性规划目标函数与限制条件
  • 求解问题并转换输出结果
  • 存储结果（insert至数据库或process save）

def lower_bound(order,j): #计算某种产品的最低产量要求,这里order是dataframe，而j是product id字符串
    #product_name = "Product" + str.upper(j) #因为数据库中的名字是ProductA之类的
    order_j = []
    for i in range(len(order)):
        if order["product_id"][i] == j:
            order_j.append([order["quantity"][i],order["days"][i]])
    low = 0
    for i in order_j:
        if (float(i[0]) != 0) & (float(i[1]) != 0):
            low += float(i[0])/float(i[1])
    return low

def upper_bound(order,j):
    #product_name = "Product" + str.upper(j) #因为数据库中的名字是ProductA之类的
    order_j = []
    for i in range(len(order)):
        if order["product_id"][i] == j:
            order_j.append([order["quantity"][i],order["days"][i]])
    up = 0
    for i in order_j :
        if (float(i[0]) != 0) & (float(i[1]) != 0):

    return up

def init3(worker_efficiency,order,m_num,n_num,s_num,worker,product,procedure,project):
    ###问题定义
    #变量
    variables = np.array([pulp.LpVariable('x%d_%d_%d'%(i,j,k), lowBound = 0, upBound = 1,cat = pulp.LpInteger) 
    for i in range(0, m_num) for j in range(0, s_num)  for k in range(0,n_num)]) #每个工人每天至多4个shift

    #目标函数
    z = sum(np.random.randint(1,2,m_num*s_num*n_num) * variables)

    #约束条件
    constraints = [] #用来存储限制条件
    res = [] #用来记录每个工人的总班次
    
    #约束条件0 每人每个shift只能有一项工作
    for j in range(s_num):
        for i in range(m_num):
            a = np.zeros([m_num,s_num,n_num])
            for k in range(n_num):
                a[i,j,k] = 1
            constraint_k = sum(a.flatten()*variables) <= 1
            constraints.append(constraint_k)
            
            
    #约束条件0.1 每一个人工作尽量向前放
    for j in range(s_num-1):
        for i in range(m_num):
            a = np.zeros([m_num,s_num,n_num])
            b = np.zeros([m_num,s_num,n_num])
            for k in range(n_num):
                a[i,j,k] = 1
                b[i,j+1,k] = 1
            constraint_k = sum(a.flatten()*variables) >= sum(b.flatten()*variables)
            constraints.append(constraint_k)

    #约束条件1 保证当日所有产品的产量超过最低要求
    for k in range(n_num):
        low = lower_bound(order,product[k])
        ek = [] #工人生产产品j的效率

#第三种排班方式的求解函数
def solve_ilp3(z, constraints):
    """
    求解问题
    """
    prob = pulp.LpProblem('LP', pulp.LpMaximize)
    prob += z
    for cons in constraints:
        prob += cons
    #print(prob)
    status = prob.solve()
    if status != 1:
        return None
    else:
        return [[v,v.varValue.real] for v in prob.variables()]
